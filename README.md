# Hack the North — Frontend Challenge

Event timeline and schedule viewer for Hack the North frontend challenge.

Codebase and documentation generated by Claude Code.

## Setup

```bash
npm install
npm run dev        # http://localhost:3000
npm run build      # production build
```

To regenerate GraphQL types after schema changes:

```bash
npm run codegen:watch
```

## Project layout

```
app/
  layout.tsx              Root layout — fetches events, wraps providers
  page.tsx                Home page (timeline)
  globals.css             Tailwind config, semantic tokens, dark overrides
  events/[id]/page.tsx    Individual event page (SSR)
  _components/            Page-scoped components
    Timeline.tsx           Event list grouped by date
    EventCard.tsx          Card for a single event
    EventDetail.tsx        Full event detail view
    EventDetailDialog.tsx  Modal wrapper for event detail
    DateLabel.tsx          Sticky date header
    FilterBar.tsx          Event type filter toggles

components/
  ui/                     Design system primitives
    Button, Card, Badge, Text, Heading, Input,
    Dialog, Icon, Container, Logo
  shared/                 Cross-page components
    Navbar.tsx             Top bar (logo, search, clock, theme toggle, auth)
    SearchDialog.tsx       Cmd+K command palette
    LoginDialog.tsx        Auth modal

lib/

  auth-context.tsx        Auth provider (localStorage)
  theme-context.tsx       Dark mode provider (useSyncExternalStore)
  events-context.tsx      Events context (avoids prop drilling to search)
  use-search.ts           Fuzzy search hook (name, speaker, description)

```

## Design decisions

**Colocation** — page-specific components live in `app/_components/`. Shared UI primitives live in `components/ui/`. Cross-page features (navbar, search, login) live in `components/shared/`.

**Server vs Client** — data fetching happens in Server Components (`layout.tsx`, `page.tsx`, `events/[id]/page.tsx`). Interactive components are `'use client'`.

**Styling** — all styling is Tailwind utility classes. Semantic tokens (`bg-surface`, `text-foreground`, `border-border`, `text-muted-foreground`) defined in `globals.css` via `@theme extend` auto-switch between light and dark mode. An inline `<script>` in the root layout prevents flash of wrong theme on load.
